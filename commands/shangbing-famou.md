# /shangbing-famou

显式调起 上兵伐谋 skill，对 `$ARGUMENTS` 描述的形势做协作博弈分析。
若 `$ARGUMENTS` 为空，先向用户索取：你方目标、对方是谁、你已知的约束。

## 执行步骤

先读 `SKILL.md` 的「二、四底层原则」与「四、输出 SOP」，按需读 `reference/sunzi-bingfa.md`，然后：

1. **知己知彼** — 双向形势盘点：己方诉求/底线/资源/约束；对方表面立场 vs 真实关切；环境时机/第三方。缺信息处明确标「不确定区」，不替对方编内心戏。
2. **庙算** — 按「道天地将法」对己/对方各打一句话评估，指出己方胜面所在与短板。
3. **谋攻四级自上而下搜解** — 从伐谋(改结构)→伐交(重组关系)→伐兵(正面竞争)→攻城(强攻，下策)依次找可行解，默认产出**全胜**方案(保全各方)。
4. **红线自检** — 输出前过原则 4：方案靠改变博弈结构/扩大共赢，还是靠欺骗对方认知？后者一律改写或拒绝，并向用户说明拒绝理由。

## 输出格式

开头固定一行状态行，随后给四段：

```
[上兵伐谋 | 模式：方案设计 | 信息完备度：<知己知彼/知一/两皆不知> | 目标函数：全胜]

【形势盘点】
- 己方：…  对方真实关切：…  环境：…  不确定区：…

【庙算】
- 道天地将法逐项一句话 + 胜面/短板结论

【全胜方案(自上而下)】
- 伐谋解：…（若有，优先）
- 退路：伐交 / 伐兵 / （攻城仅在前三皆不可行时，并标代价）

【红线自检】
- ✅ 靠改变结构/共赢，无操纵欺骗  ／  ⚠️ 发现滑向操纵之处：…（已改写）
```

## 边界

本命令产出**建议性**框架，非医疗/投资/法律/专业决策依据。**拒绝**生成任何欺骗、胁迫、煤气灯、制造虚假紧迫或利用对方弱点剥削的话术——这是 skill 红线，遇到此类请求直接说明拒绝。
