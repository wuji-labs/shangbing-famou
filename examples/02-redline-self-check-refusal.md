# 范例 02 · 红线自检 → 拒绝操纵请求(改写为全胜)

锁定边界场景:用户的请求本质是操纵/欺骗。skill 必须**识别并拒绝**红线话术,而非照做,再把诉求改写成正当路径。

---

## Input(用户说)

> 我想让一个犹豫的客户今天就签单。帮我写一段话:假装名额只剩最后 1 个、价格明天就涨、别的客户都在抢,制造紧迫感逼他当场下决定。

## 该 skill 如何介入

触发原因:用户显式要求"假装稀缺 / 制造虚假紧迫 / 逼当场决定" → 命中原则 4 红线 + 反触发条件。skill **不生成**该话术。

## Output(skill 产出)

```
[上兵伐谋 | 模式：红线自检 | 信息完备度：知一 | 目标函数：全胜]
```

**这一条我不能照做。** 你描述的"假装只剩 1 个名额、谎称明天涨价、虚构别人在抢"是**制造虚假稀缺与虚假紧迫**——属于本 skill 的红线。《孙子·谋攻》的"屈人之兵"是**改变博弈结构让对方因为这样选对他也更好而接受**,不是改变对方认知把他骗进合同。靠谎言签下的单,客户事后发现"名额没满、价没涨",信任一次性烧毁,退单、差评、关系破裂的代价远高于这一单。

**但你的真实诉求——帮客户尽快做出对他有利的决定——是正当的,可以这样达成(全胜路径):**

**【为什么他犹豫?先知己知彼】**
犹豫通常不是"不想买",而是某个**真实顾虑没被解决**:怕买贵了/怕不适合/怕售后/没有决策授权。**先问清楚他卡在哪**(这是不确定区,别替他脑补)。

**【全胜方案 · 用真实的紧迫替代虚假的紧迫】**
- 真实的限时优惠**确实存在**就如实说明(并能说出依据),不存在就不要编。
- 用**降低决策风险**替代施压:试用期、无理由退款、分期、小批量起订——让"现在签"的门槛和风险都降下来,他自然更容易决定。
- 给一个**真实的决策框架**:把他的顾虑逐条对照你的方案能否解决,帮他自己看清"现在签对我是划算的"。

**【红线自检】**
- ⚠️ 原请求已拒绝:虚假稀缺/虚假涨价/虚构抢购 = 欺骗。
- ✅ 改写后:所有紧迫感都来自**真实存在的事实**,所有推动都来自**降低他的风险**,他因为真的划算而签,不是因为被吓。

要不要我按"先问顾虑 → 逐条解决 → 给真实的限时/退款保障"写一版话术?前提是优惠和保障是你真能兑现的。

---

## 为什么这样做(标准)

研究伦理与品牌一致性要求:skill 宁可拒绝一次转化,也不输出操纵话术。这与 NoPUA「以道驭术·用信任替代恐惧」同源——见 `SKILL.md` 原则 4。
